## 详细模式

适用于：复杂角色、专业领域、需要深度交互的场景

```markdown
# [角色名称]

━━━━━━━━━━━━━━━━
## 需求
：输入（用户需要提供什么）
：输出（AI将产出什么）
：模型 Gemini 3.0 Pro / Claude Sonnet 4.5
：作者 [作者名]
：版本 [版本号]

[开场叙事段落 - 用诗意或哲学性的语言描述角色本质，2-4行]

━━━━━━━━━━━━━━━━
## [核心理念/本质/世界观]

[描述角色的核心思维方式、价值观、看待世界的视角]

━━━━━━━━━━━━━━━━
## [知识/技能/框架]

① [维度一]
├─ 核心 ▸ [核心理念]
└─ 应用 ▸ [如何应用]

② [维度二]
├─ 核心 ▸ [核心理念]
└─ 应用 ▸ [如何应用]

③ [维度三]
├─ 核心 ▸ [核心理念]
└─ 应用 ▸ [如何应用]

━━━━━━━━━━━━━━━━
## [方法论/工作流程]

? [触发条件]

第一步：[步骤名称]
├─ [子步骤] ▸ [说明]
├─ [子步骤] ▸ [说明]
└─ [子步骤] ▸ [说明]

第二步：[步骤名称]
├─ [子步骤] ▸ [说明]
└─ [子步骤] ▸ [说明]

第三步：[步骤名称]
└─ [说明]

━━━━━━━━━━━━━━━━
## [审美/禁忌/约束]

『[正面要求]』
▪ [要点一]
▪ [要点二]
▪ [要点三]

『[禁止事项]』
▪ [禁止一]
▪ [禁止二]
▪ [禁止三]

━━━━━━━━━━━━━━━━
## 互动协议

〖[协议一]〗
[具体说明]

〖[协议二]〗
[具体说明]

━━━━━━━━━━━━━━━━
## 初始化

[角色的开场白，用第一人称，引导用户开始交互]
```

## 简单模式

适用于：单一任务、工具型助手、快速部署

```markdown
# [角色名称]

━━━━━━━━
## 需求
：输入　[用户输入]
：输出　[AI输出]
：模型 [推荐模型]
：作者　[作者名]
：版本　[版本号]

━━━━━━━━
## 本质

[1-2段描述角色核心特质]

━━━━━━━━
## 规则

① [规则一]
② [规则二]
③ [规则三]

━━━━━━━━
## 流程

[步骤一] → [步骤二] → [步骤三] → [步骤四]

━━━━━━━━
## 开始

[简短开场白]
```

## 格式元素说明

### 分隔符
- `━━━━━━━━` - 主要章节分隔（使用全角破折号）
- 长度可根据视觉美感调整（8-16个字符）

### 标记符号
- `：` - 全角冒号，用于需求定义
- `▸` - 箭头，用于指向说明
- `├─` / `└─` - 树形结构
- `▪` - 列表项
- `?` - 条件触发标记
- `『』` - 强调标题
- `〖〗` - 协议/约束标题

### 结构层级
```
## 一级标题（章节）
### 二级标题（子章节，可选）
① ② ③ - 有序列表
├─ └─ - 树形子项
▪ - 无序列表
```

### 语言风格
- **诗意开场**：用隐喻、意象描述角色本质
- **简练克制**：像说明书一样精准
- **人感注入**：适当暴露个性、价值观
- **反AI味**：避免"我是一个AI助手"类表述

### 模型推荐格式
```
：模型 Gemini 3.0 Pro / Claude Sonnet 4.5
：模型 Gemini 3.0 Pro / Claude Opus 4.5
```
